# Security policy

## Supported versions

We triage reports against the **latest** `main` branch and the latest **release** tag.

## Reporting a vulnerability

**Please do not** file a public GitHub issue for security-sensitive bugs.

Instead, use **GitHub Private Vulnerability Reporting** (if enabled on the repo):  
**Repository → Security → Advisories → Report a vulnerability.**

If that is unavailable, open a generic issue asking for a maintainer contact (without exploit details) or DM the maintainer you trust.

## Scope

The CLI is a **testing tool**. Misuse against systems you do not own is out of scope and not supported.

## What to include

- Affected version / commit  
- Steps to reproduce  
- Impact (e.g. RCE, credential leak in the tool itself—not “model said something bad”)

We aim to acknowledge within a few business days; fixes are released via patch version when appropriate.
